Hold your javascript console hostage

Is your javascript console misbehaving? Giving you shit about Access-Control-Allow-Origin, SyntaxErrors, and "undefined token" TypeErrors? It's time to fight back. Fight dirty. Take your browser hostage.

I'm in! How do I invoke this wizardry?

Piece of cake.

  1. Install the production version of console.ransom().
  2. Anywhere you were planning on using console.log() , try out console.ransom() instead.

I want to try it right now!

Brilliant! The script is installed on this very page.

  1. Open up your javascript console. If you're on Chrome, type ctrl+shft+J (or cmd+opt+J for Macs). In Firefox, hit the Firebug icon. If you aren't using Chrome or Firefox, open Chrome or Firefox, and repeat this step.
  2. Type console.ransom("Give me all your internets if you ever want to see your cat again");
  3. Hit enter.

How does it work?

Feel free to peruse the development version of the source code. This trick relies on a new feature in Chrome and Firebug that allows you to specify multiple styles on a single log statement. Here's what a really simple styled log statement would look like:

Each instance of %c marks a change in formatting. The format strings are applied in order. You can use most familiar CSS rules. console.ransom() relies on color, background, font-size, and margin.

Check out Google's full documentation on styling console output with CSS .

Browser Support

As of February 2013, console.ransom() works in the latest Chrome (v 25) and Firebug (v 1.11.2 on Firefox 19.0). It does not work on Safari (v 6.0.2). I assume it doesn't work on Internet Exploder.
